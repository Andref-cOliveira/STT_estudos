Estudo arquiteturas STT GIT - reconhecimento de fala

Estudo de arquiteturas de reconhecimento de fala(speech recognition) para a lngua portuguesa (Brasil), problema conhecido tambm como Speech-To-Text.

Base de Dados - Portugus Brasil

Corpus CEFALA 1

Os arquivos de audio utilizados para o reconhecimento de fala em portugus fazem parte da base de dados Corpus CEFALA-1 do Centro de Estudos da Fala, Acstica, Linguagem e msicA (CEFALA-UFMG). O Corpus CEFALA-1 possui audios de 104 locutores diferentes. Cada locutor foi gravado com 5 microfones diferentes, o que gera 5 arquivos de audio distintos para cada locutor. Cada arquivo de audio dividido em trs partes. O primeiro segmento referente a fala espontnea, o segundo segmento refere-se a leitura de um texto longo e por ltimo segmentos referentes a leitura de 20 frases curtas. Os textos referentes a fala espontnea no so disponibilizados. O texto longo e as 20 frases podem ser encontradas no arquivo CorpusCEFALA1_textos.json. Para segmentao dos audios disponibilizado arquivos TEXTGRID. A extenso TEXTGRID pode ser visualizada com o Praat juntamente com os arquivos de audio. O Praat um software livre criado por Paul Boersma e David Weenink da Universidade de Amsterd. O pacote de Python praat-textgrid foi usado para manipulao do arquivo TEXTGRID. Esse pacote pode ser instalado com o seguinde comando:

pip install praat-textgrids

Esse pacote foi utilizado para segmentar os arquivos de audios originais com os arquivos TEXTGRID originais, e tambm para gerar novos arquivos TEXTGRID para particionar os arquivos de audio referente a leitura de texto, juntamente com a ajuda do pacote aeneas. O pacote aeneas pode ser instalado com o seguinte comando:

pip install aeneas

Para a segmentao dos audios referentes ao texto alm do pacote aeneas, utilizado tambm o arquivo CorpusCEFALA1_TEXTO.txt que divide os paragrafos com uma quebra de linha. Assim o pacote aeneas segmenta os audios em cada paragrafo.

Aps download do Corpus CEFALA1, colocar o diretrio CorpusCEFALA1 dentro da pasta data

data/CorpusCEFALA1

Para o pr-processamento necessrio tambm fazer download tambm da pasta SEG" com todos os arquivos textgrid. Colocar esta pasta dentro do diretrio data/CorpusCEFALA1"

data/CorpusCEFALA1/SEG

Obs: Pasta SEG utilizada para o pr-processamento do Corpus CEFALA1 com o notebook de jupyter.

Corpus - C-ORAL-BRASIL

O projeto C-ORAL-BRASIL visa ao estudo da fala espontnea do portugus brasileiro. O projeto tem sua sede no Laboratrio de Estudos Empricos e Experimentais da Linguagem (LEEL) da Faculdade de letras da UFMG, coordenados pelos professores Tommaso Raso e Heliana Mello.

Consiste em um conjunto de audios referentes a conversaes entre diferentes locutores. E acompanhado de arquivo que contm a transcrio dos audios, porm precisam de ser tratados para retirar informaes e o que no necessrio.

Ao invs de baixar direto do Git o repositrio, usar o repositrio dentro desta pasta com as configuraes para o treinamento em portugus.

SpeechWavenet

O modelo Speech-to-Text-WaveNet no possui artigo, somente o repositrio no Github.

No repositrio, tem-se os arquivos para fazer o treinamento com base de dados em ingls. Baseando-se nesses arquivos, foi gerado dois notebooks de jupyter com scripts em python para realizar o treinamento com a base de dados em portugus.

O modelo tbm tem alguns pr-requisitos de verso de pacotes, como o tensorflow e sugartensor. Neste caso indicado criar um novo ambiente com a verso do Python 3.5.6.

Com o conda pode ser criado com o seguinte comando:

conda create -n speech-wavenet python=3.5.6

Aps a criao do ambiente, para configurao dos pacotes do python pode-se usar o arquivo require_speechwavenet_pt.txt

pip install -r require_speechwavenet_pt.txt

Aps a configurao do ambiente ser possvel a utilizao dos scripts em python dos notebooks de jupyter.

  • CorpusCEFALA1-process.ipynb

  • Train-Speech-Wavenet.ipynb

Caso os scripts no notebook Train-Speech-Wavenet.ipynb estejam todos funcionando, pode se usar o script:

python train.py
